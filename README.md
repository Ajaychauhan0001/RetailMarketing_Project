# 🛒 RetailMart Data Engineering Pipeline

## 📌 Project Overview

The **RetailMart Data Engineering Pipeline** is an end-to-end ETL (Extract, Transform, Load) project developed to process and analyze retail sales data collected from multiple stores. The project demonstrates the complete data engineering workflow, including data ingestion, cleaning, transformation, loading into a database, SQL-based reporting, and summary generation.

This project was developed as part of a **Data Engineer Technical Assignment** to showcase practical skills in Python, Pandas, SQL, and SQLite.

---

## 🚀 Features

* Read data from multiple CSV files
* Detect and handle missing values
* Remove duplicate records
* Convert columns to appropriate data types
* Merge multiple datasets into a single DataFrame
* Calculate total revenue for each transaction
* Generate city-wise revenue reports
* Store processed data in a SQLite database
* Execute SQL queries for business insights
* Generate summary reports
* Implement reusable pipeline with error handling

## 📊 Project Workflow

### 1. Data Ingestion

* Load `sales_data.csv`
* Load `products.csv`
* Load `stores.csv`

### 2. Data Cleaning

* Remove duplicate records
* Handle missing values
* Convert data types

### 3. Data Transformation

* Merge datasets using `product_id` and `store_id`
* Create `total_revenue` column
* Generate city-wise revenue reports
* Calculate statistical metrics

### 4. Data Loading

* Create SQLite database
* Load processed data into the `retail_sales` table

### 5. SQL Reporting

* Top 3 best-selling products
* Revenue generated per store per day

### 6. Summary Report

* Total transactions
* Total revenue
* Top-selling city
* Top-selling product

### 7. Error Handling

* Implemented `try-except` blocks to handle missing files and unexpected runtime errors.
